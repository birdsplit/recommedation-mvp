# 침대 구매 의사결정 MVP 앱 기획서

> **목적**  
> 자취생·1인 가구가 침대를 구매할 때 여러 쇼핑몰과 리뷰를 오래 비교하지 않아도, 자신의 생활조건에 맞는 후보 3개를 빠르게 확인하고 구매 결정을 내릴 수 있는지 검증한다.

> **MVP 한 줄 정의**  
> 슈퍼싱글 침대 프레임 30개의 구매판단 데이터를 수동으로 구축하고, 사용자의 수납·청소·운반·조립·예산·배송 조건을 반영해 후보 3개와 추천·주의 근거를 제공하는 로그인 없는 모바일 웹 서비스

---

## 1. 기획 요약

### 1.1 해결하려는 문제

침대는 한 번 구매하면 오래 쓰고, 반품·교환·폐기가 어려운 고관여 상품이다. 자취생은 상품가뿐 아니라 수납, 청소, 운반, 조립, 배송일, 설치비, 매트리스 포함 여부, 삐걱임과 같은 리뷰 리스크까지 함께 판단해야 한다. 그러나 기존 쇼핑몰은 정보를 상품 중심으로 나열하고, 사용자의 생활조건에 맞는 결론으로 압축하지 않는다.

### 1.2 제품이 제공할 가치

- 침대 후보를 찾는 시간을 줄인다.
- 사용자가 놓치기 쉬운 조건과 추가비용을 발견하게 한다.
- 필수조건에 맞지 않는 상품을 먼저 제외한다.
- 후보를 3개로 압축한다.
- 각 상품이 나와 맞는 이유와 안 맞는 이유를 설명한다.
- 구매를 강요하지 않고 판매처에서 확인할 마지막 항목을 알려준다.

### 1.3 1차 MVP 핵심 가설

1. 이 서비스를 사용하면 기존 쇼핑 방식보다 후보 3개를 정하는 시간이 줄어드는가?
2. 이 서비스를 사용하면 자신의 조건에 맞는 상품을 찾았다는 확신이 높아지는가?
3. 추천 결과를 본 사용자가 실제 상품 상세페이지로 이동하는가?

---

## 2. 제품 원칙

1. **상품을 많이 보여주지 않는다.** 후보는 우선 3개만 제시한다.
2. **질문을 많이 하지 않는다.** 필수 질문 3개 후 결과를 먼저 보여준다.
3. **가격이 아니라 총비용을 본다.** 상품가·배송비·설치비·매트리스·추가 운반비를 구분한다.
4. **운반과 조립을 분리한다.** 사용자는 운반은 못해도 조립은 할 수 있다.
5. **단점을 숨기지 않는다.** 다만 모든 단점을 나열하지 않고 치명적 위험과 감당 가능한 위험을 구분한다.
6. **근거 없는 점수를 쓰지 않는다.** 필수조건 충족, 선호조건 충족, 확인할 위험으로 설명한다.
7. **확인된 정보와 추정 정보를 구분한다.** 출처와 마지막 확인일을 표시한다.
8. **초기에는 상담을 요구하지 않는다.** 사용자가 혼자 결정을 거의 끝낸 뒤 필요할 때만 조건을 확인하게 한다.
9. **초기에는 앱 설치와 회원가입을 요구하지 않는다.** 모바일 웹에서 즉시 이용하게 한다.
10. **수익보다 가치 검증을 우선한다.** 1차 MVP는 시간 단축과 선택 만족도를 증명한다.

## 3. 목표 사용자와 사용 상황

### 3.1 1차 핵심 사용자

- 20~39세 자취생·1인 가구
- 원룸 또는 소형 주거공간 거주자
- 3개월 내 슈퍼싱글 침대 프레임 구매 의향이 있는 사람
- 최근 1년 내 침대를 구매했거나 구매 후 후회한 사람
- 예산이 제한적이며 온라인 쇼핑을 주로 이용하는 사람

### 3.2 대표 사용 상황

- 첫 자취를 시작해 침대가 필요하다.
- 짐이 많아 수납침대와 다리형 침대 사이에서 고민한다.
- 로봇청소기를 쓰고 있어 하부 구조가 중요하다.
- 무거운 가구는 들 수 없지만 직접 조립은 가능하다.
- 이번 달 안에 받아야 한다.
- 삐걱임은 싫지만 나사 보강 정도는 할 수 있다.
- 상품가는 싸지만 배송·설치비가 얼마나 붙는지 모르겠다.

---

## 4. 전체 서비스 구조와 MVP 범위

### 4.1 장기적으로 가능한 세 가지 진입 경로

| 경로 | 사용자 상황 | 제공 가치 | MVP 포함 여부 |
|---|---|---|---|
| A. 이미 후보가 있음 | 링크·사진·상품명이 있음 | 후보 비교·구매 직전 검증 | 수동 테스트만 |
| B. 무엇을 살지 모름 | 침대는 필요하지만 종류를 모름 | 질문으로 후보 3개 압축 | **1차 구현** |
| C. 둘러보고 싶음 | 특별한 후보 없이 상품 탐색 | 특가·인기 상품 피드 | 제외 |

### 4.2 1차 MVP 구현 범위

- 경로 B만 실제 구현한다.
- 프로토타입은 상품 10개로 시작한다.
- 사용자 테스트 단계에서는 상품 30개로 확장한다.
- 매트리스가 아닌 슈퍼싱글 침대 프레임에 집중한다.
- 모든 상품 데이터는 초기에는 수동으로 등록·검수한다.

## 5. 핵심 사용자 여정

```text
유입
→ 서비스 가치 한 문장 확인
→ 필수 질문 3개 응답
→ 내가 선택한 조건 요약
→ 후보 3개 확인
→ 상품 상세 판단
→ 2~3개 비교
→ 총비용·배송·설치 확인
→ 판매처 이동
→ 결과 만족도 및 결정 확신도 응답
```

### 5.1 구매 행동 단계

1. **탐색:** 어떤 침대 구조가 필요한지 모름
2. **조건 정리:** 생활조건을 선택
3. **비교:** 후보 3개와 차이 확인
4. **최종 검토:** 총비용·배송·리스크 확인
5. **구매 이동:** 판매처로 이동
6. **구매 후 평가:** 실제 만족도 확인

## 6. 화면별 기능 기획

### 화면 1. 랜딩·시작

**목적:** 사용자가 서비스의 가치를 5초 안에 이해하고 시작하도록 한다.

**필수 요소**

- 헤드라인: “내 생활조건에 맞는 침대 3개만 골라드려요.”
- 설명: 수납·청소·운반·조립·총비용을 함께 본다는 안내
- 예상 소요시간: 약 1분
- 주요 버튼: `침대 후보 찾기`
- 보조 문구: 로그인 없이 이용 가능
- 검증용 선택 링크: `이미 후보가 있어요` → 비교 테스트 신청 또는 수동 비교 안내

**측정 이벤트**

- 랜딩 방문
- 시작 버튼 클릭
- 이미 후보 있음 링크 클릭

---

### 화면 2. 질문 1 — 침대 밑 공간

**질문:** “침대 밑 공간을 어떻게 쓰고 싶나요?”

**선택지**

- 겨울옷·이불 같은 큰 짐을 넣고 싶어요
- 자잘한 물건을 서랍에 정리하고 싶어요
- 로봇청소기가 들어가야 해요
- 먼지가 들어가지 않게 막혀 있으면 좋겠어요
- 상관없어요

**기능**

- 1개 선택
- 각 선택지에 간단한 구조 이미지 또는 아이콘
- `잘 모르겠어요` 도움말
- 선택 즉시 다음 단계 이동 또는 `다음` 버튼

---

### 화면 3. 질문 2 — 운반과 조립

**질문:** “운반과 조립은 어디까지 가능한가요?”

**선택지**

- 운반과 조립 모두 가능해요
- 운반은 어렵지만 조립은 가능해요
- 운반은 가능하지만 조립은 어려워요
- 운반과 조립 서비스가 모두 필요해요
- 친구의 도움을 받을 수 있어요

**기능**

- 운반과 조립을 하나의 설치 여부로 묶지 않음
- `운반`과 `조립`의 차이를 짧게 설명
- 엘리베이터 여부는 결과 이후 선택 정보로 받음

---

### 화면 4. 질문 3 — 예산과 배송일

**질문:** “예산과 필요한 시기는 어떻게 되나요?”

**입력 요소**

- 예산 구간: 10만 원 이하 / 20만 원 이하 / 30만 원 이하 / 상관없음
- 가격 기준: 상품가만 / 배송·설치비 포함 총비용
- 배송 시기: 이번 주 / 2주 안 / 한 달 안 / 상관없음
- 매트리스 포함 필요 여부: 선택 사항

**기능**

- 총비용 기준을 기본값으로 권장
- 아직 모르는 사용자는 건너뛸 수 있음
- 질문 완료 시 조건 요약 화면으로 이동

---

### 화면 5. 조건 요약

**목적:** 사용자가 서비스가 자신을 어떻게 이해했는지 확인하게 한다.

**예시**

> 큰 짐 수납 필요 · 운반 서비스 필요 · 직접 조립 가능 · 총비용 20만 원 이하 · 2주 안 배송

**기능**

- 조건별 수정 버튼
- `이 조건으로 3개 보기`
- 선택 항목이 충돌할 경우 안내
  - 예: 대형 수납과 로봇청소기 사용은 동시에 만족하기 어려울 수 있음

---

### 화면 6. 추천 결과 — 후보 3개

**목적:** 많은 상품이 아니라 비교 가능한 세 후보를 제공한다.

**상품 카드 필수 정보**

- 상품명·이미지·판매처
- 상품가
- 배송비
- 설치비 또는 조립 여부
- 매트리스 포함 여부
- 예상 총비용
- 예상 배송일
- 수납 방식
- 청소 방식 또는 하부 구조
- 운반·조립 조건
- 추천 수준: 매우 적합 / 조건부 적합 / 비추천

**카드 내 판단 정보**

- 가장 큰 맞는 이유 2개
- 가장 중요한 주의점 1개
- 필수조건 충족 개수
- `자세히 판단하기`
- `비교함에 담기`
- `판매처 보기`

**표시 원칙**

- 94점 같은 근거 불명확한 숫자 사용 금지
- 현재 검토 가능한 상품 수 표시
  - 예: “현재 검토한 침대 30개 중 추천”

---

### 화면 7. 상품 상세 판단

**목적:** 사용자가 추천을 납득하고 구매 전 위험을 확인하게 한다.

**섹션 구성**

1. 내 필수조건 충족표
2. 나와 맞는 이유
3. 나와 안 맞는 이유
4. 감당 가능한 리스크
5. 실질 총비용
6. 리뷰 리스크
7. 배송·설치·운반 확인사항
8. 정보 출처와 마지막 확인일
9. 최종 판단 한 문장

**예시 최종 판단**

> 수납과 먼지 관리에는 가장 적합합니다. 다만 2년 안에 이사할 가능성이 높다면 분해가 쉬운 2번 후보가 더 안전합니다.

**행동 버튼 순서**

1. `비교함에 담기`
2. `우리 집까지 총비용 확인`
3. `판매처에서 자세히 보기`

초기 화면에는 상담·전화 요청 버튼을 노출하지 않는다.

---

### 화면 8. 상품 비교

**목적:** 후보 간 핵심 차이를 한 화면에서 확인한다.

**비교 항목**

- 총비용
- 배송일
- 수납력
- 먼지 차단
- 로봇청소기 가능성
- 운반 방식
- 조립 난이도
- 매트리스 포함
- 주요 리뷰 리스크
- 이사·분해 편의성
- 최종 추천 이유

**기능**

- 최대 3개 비교
- 차이가 있는 항목 강조
- 사용자 필수조건 미충족은 빨간색 경고
- 최종 선택 버튼

---

### 화면 9. 총비용·배송 조건 확인

**목적:** 구매 직전 불확실성을 줄이되 상담 부담을 주지 않는다.

**선택 입력**

- 시·군·구 수준의 배송지역
- 엘리베이터 유무
- 계단 운반 여부
- 집 안 운반 필요 여부
- 희망 배송시기

**결과**

- 확인된 기본 배송비
- 추가비용 발생 가능 조건
- 예상 총비용 범위
- 판매처에서 최종 확인해야 할 항목

**개인정보 원칙**

- 연락처를 요구하지 않음
- 상세 주소를 받지 않음
- 비대면 판매자 확인 기능은 후속 단계로 분리

---

### 화면 10. 판매처 이동

**목적:** 추천이 실제 구매행동으로 이어지는지 측정한다.

**기능**

- 외부 판매처 링크
- 제휴 링크 또는 추적 가능한 전용 링크
- 이동 전에 핵심 확인사항 재안내
- 광고·제휴 상품 여부 명확히 표시

**측정 이벤트**

- 판매처 클릭
- 어떤 후보에서 이동했는지
- 총비용 화면을 거친 뒤 이동했는지

---

### 화면 11. 결과 피드백

**목적:** MVP 핵심 가설을 직접 검증한다.

**필수 질문**

- 이 서비스가 후보를 정하는 시간을 줄였나요? 1~5점
- 내 조건이 잘 반영됐나요? 1~5점
- 맞는 이유·안 맞는 이유가 도움이 됐나요? 1~5점
- 후보 3개 중 실제로 고려할 상품이 있나요? 예/아니오
- 다른 가구를 살 때도 사용하고 싶나요? 예/아니오
- 가장 불필요하거나 피곤했던 질문은 무엇인가요? 선택/주관식

**선택 질문**

- 최종적으로 어떤 상품을 선택했나요?

## 7. 보조 기능

### 7.1 비교함

- 최대 3개 저장
- 로그인 없이 브라우저에 임시 저장
- 비교 목록 삭제·교체

### 7.2 조건 수정

- 결과 화면에서 수납·운반·예산·배송 조건을 즉시 변경
- 변경 시 후보 재계산

### 7.3 정보 신뢰 표시

- 정보 출처
- 마지막 확인일
- 판매처 확인 필요 여부
- 추정 정보 표시

### 7.4 도움말

- 수납형과 다리형의 차이
- 운반과 조립의 차이
- 총비용에 포함되는 항목
- 로봇청소기 사용 시 확인할 하부 높이

### 7.5 오류·빈 결과 처리

- 필수조건에 맞는 상품이 없을 때 이유 설명
- 완화 가능한 조건 제안
  - 예: 배송일을 한 주 늘리면 후보 2개 추가
- 임의의 상품을 추천하지 않음

## 9. 추천 판단 규칙

### 9.1 필수조건

충족하지 못하면 후보에서 제외한다.

- 최대 예산
- 필요한 배송일
- 필수 수납 방식
- 필수 운반 서비스
- 방에 들어가는 크기

### 9.2 선호조건

충족할수록 우선순위를 높인다.

- 색상
- 콘센트
- 헤드보드
- 매트리스 포함
- 분해·이동 편의
- 빠른 배송

### 9.3 감당 가능한 리스크

사용자의 능력에 따라 감점 수준을 다르게 한다.

- 직접 나사 보강 가능
- 조립 도구 사용 가능
- 친구 도움 가능
- 배송 지연 허용
- 이사 시 분해 어려움 허용

### 9.4 결과 설명

- 필수조건 충족 수
- 선호조건 충족 수
- 확인할 위험 수
- 추천 근거 2개
- 주의점 1개
- 최종 판단 1문장

## 12. MVP 검증 방법

### 12.1 1단계 — 문제 인터뷰

- 침대 구매 예정자 또는 최근 구매자 10명
- 실제 검색 과정, 열어본 탭, 결정 지연 이유, 추가비용, 후회 요인을 확인

### 12.2 2단계 — 사용성 테스트

- 5~10명에게 화면을 직접 사용하게 함
- 이해하기 어려운 문구와 이탈 구간 확인
- 질문 수와 결과 가독성 개선

### 12.3 3단계 — 비교 실험

- 총 20~30명
- 기존 쇼핑 방식과 MVP 사용 방식을 비교
- 후보 선정 시간, 페이지 수, 결정 확신도 측정
- 순서 효과를 줄이기 위해 참가자별 수행 순서를 바꿈

### 12.4 4단계 — 구매 후 검증

가능한 사용자에게 2~4주 후 확인한다.

- 실제 구매 여부
- 배송·설치 조건 일치 여부
- 구매 후 만족도
- 놓친 조건
- 다음 가구 구매 시 재사용 의향

## 13. 구매 전환 퍼널

상담을 전면에 내세우지 않는다.

```text
추천 결과 확인
→ 상품 상세 확인
→ 비교함에 담기
→ 총비용·배송 조건 확인
→ 판매처 이동
→ 실제 구매
```

### 후속 단계에서만 추가할 기능

- 비대면 배송·설치 조건 확인
- 알림으로 답변 받기
- 사용자가 원할 때만 상담 요청
- 가장 마지막에 전화 상담 요청

### 원칙

- 연락처는 사용자의 명시적 요청이 있을 때만 수집
- 판매자에게 자동 전달하지 않음
- 초기 MVP에서는 판매처 클릭까지만 검증

## 19. 완료 기준

MVP는 다음 조건을 만족하면 구현 완료로 본다.

- 로그인 없이 시작할 수 있다.
- 사용자가 2분 안에 추천 결과를 볼 수 있다.
- 필수 질문은 3개를 넘지 않는다.
- 추천 결과는 정확히 후보 3개다.
- 모든 상품에 총비용 항목이 있다.
- 모든 상품에 맞는 이유와 안 맞는 이유가 있다.
- 모든 상품에 출처와 마지막 확인일이 있다.
- 필수조건 불충족 상품은 추천되지 않는다.
- 후보가 없을 때 조건 완화 방안을 제시한다.
- 판매처 클릭을 측정할 수 있다.
- 결과 피드백을 받을 수 있다.
- 관리자가 상품정보를 직접 수정할 수 있다.
