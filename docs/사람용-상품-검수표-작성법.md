# 사람용 상품 검수표 작성법

## 가장 먼저 알아둘 것

사람이 작성할 파일은 `data/review-checklist.ko.csv`입니다. 엑셀이나 구글 스프레드시트에서 열어도 됩니다. 첫 행의 열 이름과 `상품키_수정금지` 값은 바꾸지 마세요.

이 표는 조사와 검수 기록입니다. 이 표에서 `공개가능=예`로 바꾼 것만으로 상품이 서비스에 공개되지는 않습니다. 실제 공개 여부는 `data/catalog-products.csv`의 `status`가 결정합니다. 따라서 순서는 다음과 같습니다.

1. 사람용 표를 작성합니다.
2. `npm run review:check`로 빠진 항목을 확인합니다.
3. `npm run review:sync`로 기존 영문 검수표 형식에 반영합니다.
4. 검수 결과를 `data/catalog-products.csv`의 해당 상품 행에 옮기고 `status`를 `public`으로 바꿉니다.
5. 카탈로그 검증, 시험 임포트, 실제 임포트, 릴리스를 순서대로 실행합니다.

`review:sync`는 `review-checklist.csv`만 갱신하며 카탈로그나 데이터베이스는 바꾸지 않습니다. 사람이 확인하지 않은 값을 추측해서 채우지 않기 위한 안전장치입니다.

## 한 상품을 조사하는 순서

상품마다 `상품페이지_참고` 링크를 브라우저로 직접 엽니다. 반드시 표의 상품명과 같은 상품인지, SS 옵션이 맞는지 먼저 확인합니다. 색상, 프레임 단품/매트리스 포함, 헤드·가드 포함 여부가 다르면 같은 상품으로 보지 않습니다.

그다음 아래 순서로 확인하면 놓치는 항목이 적습니다.

1. **옵션과 가격**: SS와 정확한 색상·구성 옵션을 선택한 뒤 결제 직전 표시 가격을 적습니다. 쿠폰 적용가만 보이면 쿠폰 조건을 `공개차단이슈`에 남깁니다.
2. **재고**: 구매 버튼이 활성화되고 품절 표시가 없는지 확인합니다. 주소나 옵션을 골라야 재고가 보이면 실제로 선택해 봅니다. 확실하지 않으면 `확인불가`입니다.
3. **완성 외경**: 매트리스 규격 110×200이 아니라 프레임을 모두 조립한 뒤의 가로×길이×높이를 cm로 적습니다. 예: `114.1 x 204.9 x 120`. 높이가 가변이면 `113.4 x 213 x 81~96`처럼 사실 그대로 적습니다.
4. **수납 방식**: `없음 / 서랍 / 리프트 / 하부오픈 / 막힘 / 확인불가` 중 하나만 씁니다. 상품명만 보고 추정하지 말고 상세 이미지나 사양표에서 확인합니다.
5. **배송·설치**: 최소·최대 배송일, 방문설치 여부, 기본 배송비, 지역·엘리베이터·사다리차·층수에 따른 추가비를 확인해 한 셀에 요약합니다. 예: `4~5일, 방문설치 포함, 제주 추가비는 주문 시 확인`.
6. **조건부 비용**: 금액을 모르면 0원으로 만들지 않습니다. 카탈로그에서 `unknown`과 확인 문장으로 남길 수 있으면 `예`, 아무 설명도 없으면 `아니오`입니다.
7. **소비자 보호 정책**: 반품, 배송 중 파손 처리, 보증/A/S 세 가지가 모두 확인돼야 `예`입니다. 고객센터 번호만 있는 것은 정책 확인이 아닙니다.
8. **리뷰 표본**: 최신순에만 치우치지 않게 가능하면 최신·낮은 평점·보통 평점을 섞어 최대 10건을 읽습니다. 실제로 읽은 리뷰 수만 적고 리뷰 목록이 열린 URL을 `리뷰출처URL`에 넣습니다. 공식 상품 리뷰 화면이나 공식 리뷰 API가 명확하게 0건을 표시하면 `0`을 적을 수 있습니다. 화면을 열지 못했거나 리뷰 수를 모르는 경우에는 0건으로 처리하면 안 됩니다.
9. **반복 위험**: 아래 위험 항목별 언급 수를 `squeak:2 | smell:1`처럼 적습니다. 같은 위험이 2건 이상일 때만 추천 화면의 반복 위험으로 노출됩니다. 아무 언급도 없으면 `없음`입니다.
10. **재검수**: 다른 사람이 표본의 20% 이상을 다시 읽습니다. 10건이면 2건, 6건이면 2건, 1건이면 1건입니다. 재검수한 수와 이름을 적습니다.
11. **이미지**: 판매처 이미지의 재사용 허락이나 명시적 라이선스를 확인한 경우만 `확인`입니다. 아니면 `미확인`으로 두면 되며, 앱은 이미지 대신 자체 구조 표현을 사용합니다.
12. **최종 판단**: 해결하지 못한 사실을 `공개차단이슈`에 구체적으로 적습니다. 모든 필수 항목이 끝난 행만 차단 이슈를 비우고 `공개가능=예`로 바꿉니다.

## 리뷰에서 세는 위험 코드

카탈로그에는 아래 영문 코드만 저장됩니다. 사람용 표에는 `코드:언급수`로 적습니다.

| 코드 | 한국어로 볼 내용 |
|---|---|
| `squeak` | 삐걱거림, 소음 |
| `wobble` | 흔들림, 불안정 |
| `smell` | 냄새 |
| `assembly_hard` | 조립이 어렵거나 오래 걸림 |
| `manual_poor` | 설명서가 불명확함 |
| `missing_parts` | 부품 누락 |
| `delivery_delay` | 배송 지연 |
| `finish_poor` | 마감 불량, 찍힘, 표면 품질 문제 |
| `drawer_awkward` | 서랍 사용이 불편함 |
| `extra_cost` | 예상하지 못한 배송·설치 추가비 |

정확한 허용 코드는 `scripts/catalog-lib.mjs`의 `REVIEW_RISKS`가 최종 기준입니다. 코드가 애매하면 한국어 원문을 `공개차단이슈`에 남기고 공개 전 담당자에게 확인하세요.

## 일룸처럼 리뷰 수와 본문이 다를 때 확인하는 방법

일룸은 리뷰 요청이 세션을 요구하므로 일반 다운로드로 표본을 채우기 어렵습니다. 로그인 우회나 자동 수집을 시도하지 말고 다음처럼 확인합니다.

1. 표의 일룸 상품 링크를 일반 브라우저에서 엽니다.
2. SS와 표에 적힌 옵션을 선택합니다.
3. 페이지의 리뷰 영역을 열고, 필요하면 정상적인 로그인 절차를 거칩니다.
4. 실제로 화면에 보이는 리뷰만 최대 10건 읽습니다. 리뷰가 2건뿐이면 2건만 기록하면 됩니다.
5. 위험 언급 수를 세고, 리뷰를 본 페이지 URL과 당일 날짜를 기록합니다.
6. 다른 사람이 최소 20%를 다시 확인합니다.
7. 완성 외경은 상품 페이지에 축 이름이 없으면 공식 카탈로그 PDF의 라벨과 교차 확인합니다.
8. 평점 집계의 리뷰 수와 공식 리뷰 목록 API의 본문 수가 다르면 본문을 확보하기 전까지 공개하지 않습니다.

리뷰가 보이지 않거나 페이지가 세션 오류를 내면 `리뷰표본수=0`, `공개가능=아니오`를 유지하고 차단 이슈에 증상을 적습니다. 보이지 않은 리뷰를 평점 집계 수로 대신하거나 내용을 추정하면 안 됩니다.

2026-07-21 검수에서는 일룸 5개 중 4개를 공식 상품 페이지·공식 PDF·공식 리뷰 목록으로 확인했습니다. `iloom-HSRE01SS-3W1OS`는 현재 공식 페이지가 구매 불가 상태이며, JSON-LD 집계에는 리뷰 2건이 있지만 공식 목록 API가 0건을 반환해 본문도 확보하지 못했으므로 `sold_out` 공개 제외 상태입니다.

## 작성값 검사와 변환

PowerShell에서 프로젝트 폴더로 이동한 뒤 실행합니다.

```powershell
npm run review:check
npm run review:sync
git diff -- data/review-checklist.csv
```

첫 명령은 파일을 바꾸지 않고 열 이름, 상품키, 허용값, 리뷰 20% 재검수, 공개 가능 표시의 모순을 검사합니다. 두 번째 명령은 검사가 통과한 사람용 표를 기존 `data/review-checklist.csv` 형식으로 변환합니다. 마지막 명령으로 변환 결과가 의도와 같은지 사람이 한 번 확인하세요.

`npm run review:prepare`는 카탈로그와 기존 검수표에서 사람용 표를 새로 만드는 명령입니다. 이미 파일이 있으면 사람의 입력을 보호하기 위해 중단합니다. 정말 새로 만들 때만 기존 파일을 별도 보관한 뒤 `npm run review:prepare -- --force`를 사용하세요.

## 카탈로그 공개 반영 체크

사람용 표만으로는 카탈로그의 69개 상세 필드를 자동으로 안전하게 채울 수 없습니다. 특히 배송일 숫자, 근거 URL, 정책 요약, 리뷰 위험별 JSON, 확인일과 신뢰도를 해당 상품 행에 옮겨야 합니다. 공개 행에는 최소한 다음이 필요합니다.

- `availability=in_stock`, 정확한 현재가, 완성 외경의 가로·길이, 수납 방식, 조립 인원, 배송 최소·최대일
- 반품·파손·보증 요약
- 공식 리뷰 0건 확인 또는 리뷰 표본 1~10건, 위험별 횟수, 표본 확인일, 20% 이상 재검수
- commercial/delivery/spec/policy/review 근거 URL과 각 confidence=`confirmed`
- 상업 정보는 기준일 7일 이내, 사양·리뷰는 30일 이내
- `verified_by`, 출처 설명, `status=public`

자동 검증 결과가 오류 0개여도 내용의 사실 여부를 대신 확인해 주지는 않습니다. 브라우저에서 본 값과 CSV가 같은지 마지막으로 한 번 대조하세요.
